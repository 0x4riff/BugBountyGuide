# Glossary

- **Authorization gate** — Pre-flight check that blocks active testing until scope evidence is recorded.
- **Hypothesis ledger** — Structured list of falsifiable claims, each with one changed variable and controls.
- **Positive control** — Own authorized action succeeds as expected.
- **Negative control** — Nonexistent/invalid input behaves correctly (denied or not found).
- **Cross-account control** — Second owned account is denied access to first account's object.
- **Clean-session control** — Result reproduced without stale cookies/cache.
- **Lead** — Observation or tool output only; not a vulnerability.
- **Candidate** — Reproducible anomaly worth validation.
- **Validated** — Controls eliminate common alternative explanations.
- **Reportable** — Concrete security boundary and impact proven safely.
- **Risk budget** — Scored request volume, state mutation, data sensitivity, stability, and third-party interaction.
- **Stop condition** — Trigger that immediately halts testing (real-user data, instability, scope violation).
