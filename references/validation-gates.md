# Validation gates

## Gate 1 — authorization

Exact asset matches explicit policy. Technique allowed. Limits known.

## Gate 2 — safety

Synthetic data and owned accounts available. Test reversible. Request budget bounded. Stop condition defined.

## Gate 3 — evidence

Baseline captured. One variable changed. Positive and negative controls pass. Clean-session result reproduced.

## Gate 4 — impact

Concrete unauthorized capability shown without accessing third-party data. Status code, error, banner, or response-size difference alone fails this gate.

## Gate 5 — report

Secrets redacted. Claims match evidence. Severity not inflated. Testing stopped.
